# Hardware Design – Motor Health Monitoring System

## Overview

This repository documents the **complete hardware design** of the **Motor Health Monitoring System**.
The system is built around an **AT89C51 (8051) microcontroller** and integrates sensing, motor control, user interaction, communication, and power management blocks.

The hardware has been **fully designed and validated using Proteus simulation**.
At the time of writing, **no physical PCB has been fabricated**.

---

## Hardware Design Objectives

* Reliable motor parameter sensing
* Safe motor drive with immediate shutdown under fault conditions
* Deterministic digital and analog signal routing
* Modular and expandable circuit structure
* Educational clarity aligned with industry practices

---

## System-Level Hardware Architecture

The system is divided into the following functional blocks:

* Power input and regulation
* Microcontroller core
* Motor driver and PWM enable control
* Sensor interface block
* Display and user interface
* Communication and debug interface
* Status indication LEDs
* Test points for validation

---

## Power Architecture

### Power Sources

* **USB power input** (logic power and debugging)
* **External 12V input** (motor supply)

### Regulation

* **7805 Linear Voltage Regulator**

  * Input: 12V
  * Output: +5V (MCU and peripherals)
* Decoupling capacitors:

  * 0.1µF (high-frequency noise suppression)
  * 10µF (bulk stabilization)
* Capacitors placed close to the regulator and IC power pins

### Power Distribution

* +5V rail for MCU, sensors, OLED, and logic circuits
* Motor supply isolated through the motor driver
* Common ground reference across all hardware blocks

---

## Microcontroller Core

### MCU

* **AT89C51 (8051 family)**
* External **12 MHz crystal oscillator**
* RC-based reset network
* **EA pin tied HIGH** to enable internal program memory

### Key Responsibilities

* Sensor data acquisition
* Fault detection and latching
* PWM generation for motor speed control
* Display updates and UART communication
* Global motor enable/disable control

---

## Motor Driver and Control

### Motor Driver

* **L293D Dual H-Bridge Motor Driver**
* Separate logic and motor supply pins
* Internal flyback diode protection

### PWM Enable Circuit

* PWM signals generated by the MCU
* Enable pins routed via pull-up and series resistors
* Allows:

  * Speed control
  * Immediate motor shutdown during faults

### Protection Strategy

* Any detected fault disables the motor globally
* Fault condition remains latched until system reset

---

## Sensor Interface Block

### Current Sensor

* **ACS712 Hall-Effect Current Sensor**
* Measures motor current
* Analog output routed to ADC
* Used for overcurrent fault detection

### Temperature Sensor

* **LM35 Precision Temperature Sensor**
* Linear analog output (10 mV/°C)
* Used for thermal protection

### Vibration Sensor

* **SW-420 Vibration Sensor Module**
* Comparator-based digital output
* Detects abnormal mechanical vibrations

### IR Reflective Sensor

* Used for speed or object detection
* Digital output connected to MCU GPIO

---

## Analog-to-Digital Conversion

* External ADC used for analog sensor interfacing
* Supports multiple sensor channels
* SPI-like digital communication with MCU
* Improves measurement accuracy and noise immunity

---

## Display Interface

### OLED Display

* **0.96-inch SSD1306 OLED**
* I²C communication protocol
* Displays:

  * Motor status
  * Fault information
  * Sensor readings
* External pull-up resistors used on SDA and SCL lines

---

## User Interface

### Keypad

* **4×4 Matrix Keypad**
* Row-column scanning technique
* External pull-up resistors provided
* Used for:

  * Menu navigation
  * Manual reset
  * Parameter selection

---

## Communication Interfaces

### UART Debug Port

* TTL-level UART interface
* Used for:

  * Debug logging
  * Sensor monitoring
  * Fault reporting
* Compatible with USB–TTL converters

### USB Interface

* USB module used for power and communication support
* Includes basic filtering and ESD protection

---

## Status Indication LEDs

* Power ON indicator
* Motor ON indicator
* Fault indicator
* Series resistors selected to limit LED current safely

---

## Test Points and Debug Support

Dedicated test points are provided for:

* Power rails
* UART TX and RX
* PWM outputs
* Sensor signals

These enable:

* Easy probing during simulation
* Oscilloscope-based validation
* Logic analyzer connections
